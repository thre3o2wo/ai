Ch. 9 브라우저 객체

지금까지 배운 JavaScript는 '언어' 자체였다면, 이번 챕터는 그 언어를 이용해 '브라우저(Browser)' 자체를 제어하는 방법을 다룬다. HTML 문서를 제어하는 '문서 객체 모델(DOM)'과는 구별되는 개념.

---

1. 브라우저 객체 모델 (BOM)이란?

- 개념: JavaScript가 웹 브라우저의 창, 주소창, 방문 기록 등 브라우저의 '기능'에 접근하고 제어할 수 있도록 묶어놓은 객체들의 집합.

---

2. window 객체 : 모든 것의 최상위 객체

BOM의 최상위(Root) 객체이며, JavaScript의 '전역(Global) 객체'.

- 키워드: window, 전역 객체, 이벤트 속성
- 특징:
    - 모든 전역 변수와 함수는 window의 속성: Ch.1에서 배운 alert, prompt, confirm, setTimeout 등은 모두 window 객체에 속한 '메서드'. (window.alert()가 정식 명칭)
    - 1_window객체의속성및메소드.html 실습에서 var chk = ...로 선언한 변수가 for...in 루프에서 window.chk로 발견되는 것이 그 증거.
    - 이벤트 핸들러: window.onclick, window.onload처럼 on...으로 시작하는 '이벤트 속성'에 함수를 할당하여 브라우저의 전역 이벤트를 처리할 수 있다.

---

3. window 객체의 메서드 : 새 창 제어

window 객체는 팝업창을 열고 닫는 등, 창 자체를 제어하는 메서드를 가진다.

- 키워드: window.open(), window.close(), target
- window.open(URL, 이름, 옵션):
    - 3_새창열기.html 실습처럼 새 창(팝업창)을 연다.
    - URL: 새 창에 열 주소. ('' 비워둘 수 있음)
    - 이름: 4_open에윈도우이름을쓰는경우.html 실습이 핵심. open('', 'childWin', ...)처럼 '이름'을 지정하면, <a href="..." target="childWin"> 태그가 바로 그 'childWin'이라는 이름의 창을 제어할 수 있게 된다.
    - open() 함수는 child 변수처럼 '자식 창 객체'를 반환하며, 부모 창은 child.document.write()나 child.close() 등으로 자식 창을 직접 제어할 수 있다.
- window.close():
    - 5_child(large).html 실습처럼, setTimeout 등을 이용해 자식 창이 스스로를 닫게 할 수도 있다.

---

4. location 객체 : 주소창 제어

브라우저의 주소 표시줄(URL)을 제어하는 객체.

- 키워드: location.href, location.assign(), location.replace()
- 역할: 페이지를 다른 URL로 이동(Redirect)시킬 때 사용한다.
- 실습 적용: 6_location.html 파일은 setTimeout을 이용해 3초 후에 location.assign('7_history.html')을 호출하여 '7_history.html' 페이지로 이동시킨다.
- assign() vs replace():
    - location.href나 location.assign()은 방문 기록(History)을 남긴다. (뒤로 가기 가능)
    - location.replace()는 방문 기록을 남기지 않는다. (뒤로 가기 불가능)

---

5. history 객체 : 방문 기록 제어

브라우저의 '뒤로 가기', '앞으로 가기' 버튼과 관련된 객체.

- 키워드: history.back(), history.go()
- 역할: 사용자의 방문 기록을 탐색한다.
- 실습 적용: 7_history.html 실습에서 history.back() 또는 history.go(-1)을 onclick 이벤트에 연결하여, 브라우저의 '뒤로 가기' 버튼과 동일한 기능을 구현한다.
