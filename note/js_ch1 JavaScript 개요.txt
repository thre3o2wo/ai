HTML/CSS가 '구조(뼈대)'와 '디자인(옷)'이었다면, JavaScript는 '동작'을 담당하는 핵심 파트.

---

1. JavaScript란 무엇인가? (역할)

- 동적인 웹페이지: JavaScript의 존재 이유. 정적인 HTML 문서를 '살아 움직이게' 만든다. (예: 마우스 클릭에 반응하기, 내용 동적으로 바꾸기, 팝업창 띄우기).
- 브라우저 종속성: JavaScript 코드는 HTML에 종속적이며, 브라우저(크롬, 엣지 등)에 내장된 'JavaScript 엔진'이 코드를 해석하고 실행한다.
    - (참고: 0.js 차례.txt의 node.js 언급처럼, 크롬의 V8 엔진을 브라우저 밖으로 꺼내 단독으로 실행할 수 있게 만든 것이 node.js이며, 이는 나중에 '백엔드' 개발에서 만나게 됨.)

---

2. <script> 태그와 실행 순서 (위치)

- <script> 태그: HTML 안에서 JavaScript 코드를 작성하거나, 외부 .js 파일을 불러올 때 사용.
- 실행 순서 (중요): 브라우저는 HTML 문서를 위에서 아래로 순차적으로 읽고 실행한다.
    - 1_firstEx.html 실습 참고. <head>의 "첫 번째 메시지"가 먼저 실행되고, <body>의 "두 번째 메시지"가 표시된 후, "세 번째 메시지"가 실행되는 순서.
- 외부 파일 분리 (<script src="...">):
    - 3.html 실습처럼, <script src="3.js"></script>를 사용해 CSS처럼 JS 코드를 별도 파일로 분리하는 것이 실무 표준.

---

3. 기본 출력 방법과 디버깅 (결과 확인)

JavaScript 코드가 잘 작동하는지 확인하는 방법들.

- document.write() (옛날 방식):
    - 페이지가 로드되는 시점에 HTML 문서 본문에 직접 텍스트나 HTML 태그를 씀.
    - (주의: 페이지 로드가 끝난 후 사용하면 기존 문서를 다 지우기 때문에, 지금은 innerHTML 이나 textContent를 더 선호함. 1031.ch01_개요.docx 의 innerHTML 예제가 더 좋은 방식.)
- alert() (팝업창):
    - 사용자에게 알림 팝업창을 띄운다. 코드 중간에 넣어 실행 흐름을 멈추고 확인할 때 유용함.
- console.log() (핵심 디버깅 도구):
    - 가장 중요. 사용자 화면에는 보이지 않고, F12(개발자 도구)의 'Console' 탭에 변수 값이나 메시지를 출력한다.
    - 파이썬의 print()와 똑같은 역할이며, 앞으로 alert나 document.write보다 디버깅을 위해 압도적으로 많이 사용하게 될 것.
- 오류 확인 (F12):
    - 코드가 멈추거나 이상하게 작동할 때, F12를 눌러 'Console' 탭을 확인하는 것이 디버깅의 첫걸음.
    - 2_오류확인.html 실습에서 alert('al_pha 값은 ' + al-pha); 부분은 al_pha라는 변수가 아닌 'al 빼기 pha'라는 연산으로 인식되어 오류(ReferenceError)가 발생한다.
    - 콘솔을 확인하면 이 오류가 몇 번째 줄에서 났는지 알려주며, 스크립트는 오류가 난 지점에서 실행을 멈춘다. (그래서 alert(3)이 실행되지 않음.)

4. 변수 선언: var vs let (핵심 문법)

- var (옛날 방식, 전역 변수):
    - var al_pha = 20;처럼 변수를 선언한다.
    - var로 선언된 변수는 스코프(범위)가 함수 단위이거나 전역(Global)적이어서, for 문 같은 블록({}) 안에서 선언해도 밖에서 접근이 가능하다.
    - 3.js에서 for (var i=1; ...)로 선언한 i 변수가, 3.html에서 document.writeln("최종 i 값은 " + i);로 접근이 가능한 이유가 바로 이 var의 전역적인 특성 때문임. (현재는 이런 혼란 때문에 var 사용을 지양한다.)
- let (현대 방식, 지역 변수):
    - let sum = 0;처럼 선언.
    - let은 {} 블록 스코프(지역 변수)를 가진다. for 문 안에서 let으로 선언했다면 밖에서 접근이 불가능하다.
    - (참고: 3.js의 sum 변수는 for 문 *밖*에서 let으로 선언했기 때문에, 3.html에서도 접근이 가능하다.)
- const (상수):
    - let과 같지만, 한 번 값을 할당하면 절대 바꿀 수 없는 '상수'를 선언할 때 사용.
