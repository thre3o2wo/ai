Ch. 2 기본 문법

1. JavaScript의 자료형 (Data Types)

Python과 달리, JavaScript는 '느슨한 자료형(Loose Typing)' 언어. 변수 하나가 문자열이었다가, 숫자가 될 수 있다.

- 주요 자료형:
    - string (문자열) : '와 "를 혼용할 수 있다.
    - number (숫자) : 정수와 실수를 구분하지 않는다.
    - boolean (불리언): true 또는 false.
    - object (객체) : Python의 딕셔너리와 유사한 {}  또는 리스트와 유사한 [] (배열) 모두 'object' 타입으로 취급한다.
    - function (함수): JavaScript에서는 함수 자체가 하나의 자료형.
    - undefined: 값이 할당되지 않은 변수의 기본 상태.
- typeof 연산자: 변수의 현재 자료형을 문자열로 알려준다. (예: "string", "number", "object") .

2. 변수 선언: var vs let vs const (중요)

변수를 선언하는 키워드이며, '스코프(Scope, 범위)'에서 큰 차이가 있다.

- var (옛날 방식): 전역 변수 또는 함수 레벨 스코프. 코드 블록({})을 무시하며, 변수 재선언이 가능하다. (현재는 사용을 권장하지 않음.)
- let (현대 방식): 지역 변수. {} 블록 레벨 스코프를 따르며, 변수 재선언이 불가능하다.
- const (상수): let과 스코프는 같지만, 한 번 할당하면 값을 재할당할 수 없다.

> 💡 Gemini 학습 조언: 특별한 이유가 없다면, 기본적으로 const를 사용하고, 값이 변경되어야 하는 변수만 let을 사용하는 것이 좋다. var는 일단 잊어도 된다?

3. 연산자 (Operators)

- 복합 대입 연산자 (+=): sum += i는 sum = sum + i와 같다. 2_복합대입연산자.html 실습처럼 문자열을 합칠 때도 유용하게 사용됨.
- 삼항 연산자 (? :): 간단한 if-else문을 한 줄로 줄여 준다. 4_prompt.html에서 "잘했다"/"분발해"를 구분할 때 사용함.
    - (조건) ? (참일 때 값) : (거짓일 때 값)
- 일치 연산자 (== vs ===) (매우 중요):
    - == (동등 연산자): 값만 비교한다. 0 == false가 true. (타입이 다르면 강제로 타입을 변환해서 비교)
    - === (일치 연산자): 값과 타입을 모두 비교한다. 0 === false는 false.
    - 💡 Gemini 학습 조언: JavaScript에서는 버그를 줄이기 위해 항상 ===를 사용하는 습관을 들이는 것이 좋음. 3_confirm.html의 주석처럼 ==를 쓰면 의도치 않은 결과가 나올 수 있다.

4. 자료형 변환 (Type Conversion)

JavaScript는 상황에 따라 자동으로 형 변환을 하지만, 명시적으로 변환하는 함수들이 중요.

- 숫자로 변환:
    - Number(값): 문자를 숫자로 변환한다. Number("100점")처럼 변환이 불가능하면 NaN (Not a Number)을 반환한다. Number("")는 0을 반환.
    - parseInt(값): 문자를 정수로 변환한다. parseInt("100점")처럼 숫자 뒤에 문자가 오면, 숫자 부분(100)까지만 인식한다. parseInt("")는 NaN을 반환한다.
- NaN과 isNaN():
    - NaN은 "숫자가 아님"을 뜻하는 특수한 number 타입 값.
    - isNaN(값): 해당 값이 NaN인지 판별하는 함수. 4_prompt.html에서 유효한 숫자인지 검사할 때 사용함.
