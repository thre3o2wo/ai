Ch. 6 객체

지금까지 배운 변수, 조건문, 함수, 배열이 '부품'이었다면, '객체'는 이 모든 부품을 조립하여 '하나의 완성품'을 만드는 방법. Python의 딕셔너리(dict)와 클래스(class)의 특징을 합쳐 놓은 것과 비슷하다.

---

1. 객체란 무엇인가? (속성과 메서드)

객체({})는 "서로 연관된 변수와 함수를 그룹화한 것".

- 속성 (Property): 객체 내부의 '변수'. (예: name: '홍길동') .
- 메서드 (Method): 객체 내부의 '함수'. (예: eat: function(...)).
- this (핵심 키워드):
    - 메서드 내부에서 "객체 자기 자신"을 가리킨다.
    - 02_객체의속성과메서드.html 실습에서 this.name은 "나(this)의 name 속성"을 의미한다.
    - 객체 메서드에서 자기 속성을 참조할 때는 this를 생략할 수 없다.

---

2. 객체 다루기 (순회와 조작)

객체는 배열과 다르게 '순서(index)'가 아닌 '키(key)'로 데이터를 관리한다.

- 속성 접근:
    1. 점 표기법 (.): person.name (주로 사용)
    2. 대괄호 표기법 ([]): person['age'] (키 이름에 변수를 사용하거나 특수문자가 있을 때 사용)
- for...in (객체 순회):
    - 객체를 순회하는 유일한 반복문. for...of는 배열 전용이다.
    - for (let key in obj) 구문에서 key 변수에는 '값'이 아니라 '키 이름' (예: "name", "age")이 문자열로 들어온다.
    - 따라서 obj[key]처럼 대괄호 표기법을 사용해야 값에 접근할 수 있다. (obj.key로 쓸 수 없다.)
- 속성 추가/삭제 (동적):
    - JavaScript 객체는 유연해서 언제든 속성을 추가(obj.address = '서울시')하거나 삭제(delete(obj.age))할 수 있다.

---

3. 메서드 재정의 (Method Overriding)

02_객체의속성과메서드.html에서 document.writeln(person)을 실행했을 때 [object Object]라고 출력되는 문제를 해결하는 방법이다.

- toString() 메서드:
    - 모든 객체는 toString이라는 기본 메서드를 가지고 있다.
    - document.write(person)처럼 객체를 문자열로 출력하려 할 때, JavaScript는 자동으로 person.toString()을 호출한다.
    - 03_toString.html과 04_객체배열.html 실습처럼, toString 함수를 덮어써서(Overriding) 원하는 형식의 문자열(예: key : value)이 반환되도록 만들 수 있다 . (Python의 __str__와 같다.)

---

4. 실전 응용 (데이터 관리)

객체를 왜 쓰는지 보여 주는 중요한 실습.

- 객체의 배열 (Arrays of Objects):
    - 04_객체배열.html에서처럼 students = [{'name':...}, {'name':...}] 형태로 만드는 것은, (Python의) '딕셔너리가 담긴 리스트'와 동일하다.
    - 이는 데이터베이스나 API에서 가져온 표 형태의 데이터를 다루는 가장 표준적인 자료구조.
- 두 가지 객체 생성 방식:
    1. 반복문으로 메서드 추가: 04_객체배열.html에서는 객체 배열을 만든 *후에*, for...in 반복문으로 students[idx].getSum = ...처럼 나중에 메서드를 동적으로 추가함.
    2. 함수로 객체 생성 (Factory Function): 05_객체배열(함수이용).html의 방식이 더 세련됨. regStudent라는 '객체 공장 함수'를 만들어, name, kor 등을 인자로 받으면 속성과 메서드가 모두 포함된 resultObj 객체를 '찍어내서' 반환(return)함.

이 객체 개념은 나중의 JSON(데이터 형식), Node.js, React에서 데이터를 다루는 '기본 중의 기본'.
