Ch. 4 반복문

Ch.3 '조건문'이 코드의 흐름을 '분기'시키는 것이었다면, Ch.4 '반복문'은 특정 코드를 '반복' 실행하는, 프로그래밍의 또 다른 큰 축.

특히, 데이터를 묶어서 다루는 '배열(Array)'과 이 배열을 순회(iteration)하는 다양한 for문들을 비교하는 것이 이번 챕터의 핵심.

---

1. 배열 (Array) : 데이터 묶음

가장 먼저, 반복문을 왜 써야 하는지 알려주는 '데이터 묶음' 자료형.

- 배열이란?: 하나의 변수에 여러 개의 값을 순서대로 저장하는 자료형.
- 특징: JavaScript의 배열은 Python의 리스트처럼 모든 자료형을 한곳에 담을 수 있다 (숫자, 문자열, function, object 등).
- 주요 속성 및 메서드:
    - .length: 배열의 길이를 반환.
    - .push(value): 배열의 맨 뒤에 값을 추가.
    - .pop(): 배열의 맨 뒤에 있는 값을 제거하고 그 값을 반환.
    - .unshift(value): 배열의 맨 앞에 값을 추가.
    - .includes(value): 배열에 특정 값이 포함되어 있는지 true/false로 확인.

---

2. 기본 반복문 (While / For)

가장 고전적이고 기본적인 반복문.

- while (조건식):
    - 조건식이 true인 동안 { } 안의 코드를 계속 반복 실행.
    - quiz.html 실습에서 while(true)를 사용해 '무한 루프'를 만들고, confirm 창을 이용해 빠져나오는(break) 방식을 구현함.
- do...while (조건식):
    - while문과 달리, 코드를 최소 한 번은 무조건 실행한 뒤에 조건을 검사한다 .
- for (고전적 방식):
    - for (①초기값; ②조건식; ④증감식) { ③명령문 } 의 형태로, 정해진 횟수만큼 반복할 때 가장 많이 사용.
    - 3_중첩for문.html의 구구단 예제처럼, for문 안에 for문을 넣는 '중첩 반복문' 을 만들 수 있다.

---

3. 배열을 순회하는 4가지 방법 (핵심)

2_배열출력.html에서 이 4가지 방식을 비교한 것이 이번 챕터의 하이라이트.

1. 일반 for문 (C/Java 스타일):
    - for (let idx=0; idx < arr.length; idx++) { ... }
    - 가장 기본적이며, 인덱스(idx)를 직접 제어할 수 있다.
2. for...in:
    - for (let idx in arr) { ... }
    - 배열의 '인덱스(index) 값' 또는 객체의 '키(key)' 값을 순회한다. (값이 아닌 인덱스가 idx에 들어옴.)
3. for...of (Python 스타일):
    - for (let data of arr) { ... }
    - for...in과 달리, 인덱스가 아닌 배열의 '값(value)' 자체(data)를 순회한다. (Python의 for item in list:와 가장 유사함.)
4. forEach() (추천 / 함수형 스타일):
    - arr.forEach(function(data, idx){ ... });
    - 배열(Array) 객체에만 사용 가능한 메서드(함수).
    - 배열의 각 요소를 순회하며, 인자로 넘겨준 '콜백 함수(callback function)'를 실행한다.
    - data(값)와 idx(인덱스)를 모두 받아올 수 있어 유연하고 가독성이 좋음.

---

4. 반복 흐름 제어

반복문을 중간에 탈출하거나 건너뛸 때 사용한다.

- break:
    - 반복문을 즉시 종료하고 빠져나옴.
    - quiz.html에서 사용자가 confirm 창에 '취소'를 눌렀을 때(if(!answer)) while 무한 루프를 탈출하는 데 사용함.
- continue:
    - 현재 실행 중인 반복을 즉시 중단하고, 다음 반복으로 건너뛴다.

5. quiz.html 종합 실습 분석

Ch.2~4의 핵심 개념을 모두 사용한 실습.

7. sum += score 복합 대입 연산자로 값 누적하기.
