Ch. 3 HTML-II : form, 공간 분할 태그

1. 폼 태그 (<form>) : 데이터 입력과 전송

사용자가 아이디, 비밀번호 등을 입력하고 서버로 전송할 수 있는 '입력 양식' 영역을 만듭니다.

- <form> 속성:
    - action: 입력된 데이터를 전달할 서버(페이지) 주소를 지정합니다.
    - method: 데이터를 전송하는 방식을 지정합니다.
        - get: URL 주소 뒤에 ?id=111&pw=222처럼 데이터를 붙여서 전송합니다. (기본값)
        - post: 데이터를 URL에 노출시키지 않고 숨겨서 전송합니다.
- 주요 입력(Input) 태그:
    - <input>: type 속성에 따라 모양과 기능이 달라집니다.
        - 텍스트: type="text" (일반 글자), type="password" (입력 내용 가려줌).
        - 선택: type="radio" (여러 개 중 하나만 선택, name 속성 통일), type="checkbox" (여러 개 중복 선택).
        - 특수 입력: type="file" (파일 첨부), type="date" (날짜 선택), type="color" (색상 선택), type="range" (슬라이더 바).
        - 버튼: type="submit" (폼 데이터 전송) , type="reset" (폼 내용 초기화), type="button" (JavaScript 연결용 빈 버튼).
    - <textarea>: 여러 줄의 긴 텍스트를 입력받습니다.
    - <select> / <option>: 드롭다운 목록을 만듭니다.

2. 폼 유효성 검사 (Validation)

사용자가 데이터를 올바르게 입력했는지 확인하는 과정입니다. quiz.html과 4_pw유효성검사.html 등에서 두 가지 방식을 모두 연습했습니다.

- JavaScript 사용 (스크립트 방식):
    - <form> 태그에 onsubmit="return chk()"처럼 자바스크립트 함수를 연결합니다.
    - chk() 함수가 입력값의 길이(value.length)나 비밀번호 일치 여부(pw.value != pwch.value) 등을 검사합니다.
    - 조건에 맞지 않으면 alert() (경고창) 을 띄우고 return false;를 실행해 제출(submit)을 중단시킵니다.
- HTML5 속성 사용 (브라우저 내장 기능):
    - required: 필수 입력 필드로 지정합니다.
    - placeholder: 입력창에 미리보기 힌트 텍스트를 표시합니다.
    - pattern: 정규표현식을 이용해 전화번호(\d{3}-\d{3,4}-\d{4})나 이메일 형식 등을 검사합니다.
    - autofocus: 페이지 로드 시 해당 입력창에 커서를 자동으로 위치시킵니다.

3. 공간 분할 태그 (Layout)

웹페이지의 구조와 레이아웃을 잡기 위해 영역을 나누는 태그입니다.

- 기본 태그 (div / span):
    - <div>: 블록(Block) 형식 태그. 한 줄 전체를 차지하며, 줄 바꿈이 일어납니다. (예: <p>, <ul>, <h1>)
    - <span>: 인라인(Inline) 형식 태그. 자기 내용만큼의 공간만 차지하며, 줄 바꿈이 없습니다. (예: <a>, <b>)
- 레이아웃 구성 방식:
    1. 전통 방식 (div + id): 의미 없는 <div> 태그에 id를 부여하여 (<div id="header">, <div id="content">) 레이아웃을 구성합니다.
    2. HTML5 방식 (시맨틱 태그): '의미론적인' 태그를 사용합니다. 검색 엔진이 웹페이지의 구조를 더 잘 이해할 수 있게 하고, 코드 유지보수가 쉬워집니다.
        - <header>: 머리말 영역.
        - <nav>: 메뉴 (내비게이션) 영역.
        - <section>: 본문의 주요 내용 영역.
        - <footer>: 꼬리말 (회사 정보, 저작권 등) 영역.
        - <article>: 독립적인 글(기사, 포스트) 영역.
