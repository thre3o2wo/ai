Ch. 1 웹표준 개론

1. 웹(Web)의 기본 개념
	- 웹(Web)이란? WWW(World Wide Web)의 줄임말로, 인터넷에 연결된 컴퓨터를 통해 정보를 공유할 수 있는 '정보 공간'입니다.
	- 인터넷 vs 웹: 인터넷은 전 세계를 연결하는 거대한 '국제 정보 통신망'이고, 웹은 그 인터넷 위에서 작동하는 '서비스(기능)' 중 하나입니다.
	- 핵심 용어:
    	- 프로토콜(Protocol): 네트워크에서 통신하기 위한 '약속' 또는 '규약'입니다. (예: HTTP - 웹페이지를 볼 때, FTP - 파일 전송)
    	- IP / DNS: IP는 네트워크상에서 컴퓨터를 식별하는 '주소' 이고, DNS는 이 IP 주소를 www.lec.com처럼 외우기 쉬운 '문자열'로 맵핑(연결)해주는 것입니다.

---

2. 웹 표준의 3요소 (HTML, CSS, JavaScript)

Ch.1의 가장 중요한 핵심은 이 세 가지의 역할을 명확히 구분하는 것입니다.

	 1) HTML (HyperText Markup Language):
   	 - 역할: 구조와 정보(내용)를 담당합니다.
  	 - 설명: 웹페이지의 뼈대를 잡고 "여기는 제목", "이것은 문단", "이것은 이미지"라고 의미를 부여하는 '태그'로 구성된 언어입니다.
	2) CSS (Cascading Style Sheets):
    	- 역할: 표현(디자인)을 담당합니다.
    	- 설명: HTML로 만든 뼈대에 색상을 입히고, 글자 크기를 조절하며, 레이아웃을 배치하는 등 '디자인'을 전담합니다.
	3) JavaScript (JS):
    	- 역할: 동작(Behavior)을 담당합니다.
    	- 설명: 웹페이지가 '동적'으로 움직이게 합니다. (예: 버튼 클릭 시 팝업창 띄우기, 사용자 이름 입력받기 등)

3. 실습 파일 비교: 왜 2개로 만들었나? (중요)

두 실습 파일(1.html과 2(css_js분리).html)은 이 3요소를 어떻게 '조립'하는지 보여주는 좋은 예시입니다.

	실습 1: 1.html (내부 스타일 및 스크립트)

	- 구조: HTML 파일 안에 <style> 태그(CSS)와 <script> 태그(JS)가 모두 포함되어 있습니다.
	- CSS: <style> 태그 안에 background-color: lightyellow; (배경색)과 font-size: 32pt; (글씨 크기)가 정의되어 있습니다.
	- JS: <script> 태그 안에 prompt로 사용자 이름을 입력받고 document.write로 환영 메시지를 띄웁니다.
	- 문제점: HTML(정보), CSS(디자인), JS(동작)가 한 파일에 섞여있어, 나중에 수정하고 관리하기(유지보수) 어렵습니다.
    

	실습 2: 2(css_js분리).html + 2.css + 2.js (외부 파일 분리)

	- 구조: 이것이 '웹 표준'에서 권장하는 방식입니다.
    	1. HTML (2(css_js분리).html): 뼈대만 남기고, <head> 태그 안에서 <link>를 이용해 CSS 파일을, <script src=...>를 이용해 JS 파일을 불러옵니다.
    	2. CSS (2.css): 디자인 코드(background-color, font-size)가 별도 파일로 분리되었습니다.
    	3. JS (2.js): 동작 코드(prompt, document.write)가 별도 파일로 분리되었습니다.
		- 장점: 정보(HTML), 디자인(CSS), 동작(JS)이 각자의 파일로 명확히 분리되어 있어, 웹 문서가 가벼워지고 유지보수가 훨씬 쉬워집니다.
